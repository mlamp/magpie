# M4 — DHT (BEP 5)

**Status**: planned
**Gate summary**: a `magpie-bt-dht` subcrate ships a Kademlia DHT that bootstraps from cold state, answers all four BEP 5 RPCs (ping / find_node / get_peers / announce_peer), integrates with magpie's `UdpDemux` for shared-socket dispatch, and discovers peers for a controlled loopback swarm without any tracker. Private torrents must not announce or query. Rate limits hold under inbound flood.

## Scope principle

magpie M4 ships a **standalone, well-tested Kademlia implementation** consumable from magpie-bt or independently. It is not bundled with client integration in a consumer's repo — `Engine::attach_dht(Arc<Dht>)` is the seam, and consumers feed discovered peers into `add_peer` per the same pattern PEX uses (ADR-0020 era).

## Goal

Close the biggest remaining functional gap for magpie as a general-purpose torrent library: trackerless peer discovery. Modern swarms are ≥80% DHT-populated; magpie cannot currently fall back when trackers fail, making it unusable for any deployment that expects real-world reachability.

Design is locked across three ADRs (0024–0026) before execution; this milestone is pure implementation + gating.

## Scope / deliverables

### A. Crate skeleton + wire types

- [ ] New subcrate `crates/magpie-bt-dht/` per ADR-0001.
- [ ] `NodeId([u8; 20])` with XOR-distance comparator, `Bucket`, `RoutingTable` per ADR-0024.
- [ ] KRPC message types (query / response / error) + bencode encode/decode, structured after `magpie_bt_wire::extension` (thiserror-typed error, negative/boundary tests).

### B. Transport integration

- [ ] `Dht::run(Arc<UdpDemux>, local_port)` — registers a first-byte-`b'd'` dispatch on the existing demux (reservation already in ADR-0015).
- [ ] Inbound query path: parse, route to handler, emit reply.
- [ ] Outbound query path with txid tracking (mirrors the tracker `register_tracker_response` pattern but on its own map; DHT txids are 2-byte).

### C. RPC handlers (ADR-0024 routing table)

- [ ] `ping(target)` — response; table insert on reply.
- [ ] `find_node(target)` → `r::nodes` (8 closest nodes, 26 bytes each).
- [ ] `get_peers(info_hash)` — branches on local peer table presence; `r::values` or `r::nodes`, always `r::token`.
- [ ] `announce_peer(info_hash, port, token)` with token validation per ADR-0026.

### D. Bootstrap (ADR-0025)

- [ ] Three-source contact list (cache + DNS + consumer-supplied).
- [ ] 60-s bootstrap ping round, 8 contacts per round.
- [ ] Exit criterion: ≥ 32 good nodes AND ≥ 1 non-empty `find_node` reply.
- [ ] Stalled-bootstrap alert after 10 min below 4 good nodes.
- [ ] Persistent top-64 contact cache on shutdown, load on startup.

### E. Tokens, BEP 42, rate limits (ADR-0026)

- [ ] Dual-secret rotation every 15 min; stateless HMAC-SHA1.
- [ ] BEP 42 IP-salted node ID with two-phase public-IP derivation.
- [ ] Three-tier token-bucket rate limits (per-IP inbound, global inbound, per-remote outbound).
- [ ] Silent drop on over-cap inbound (no reflection amp) + `queries_dropped_rate_limited` counter.

### F. Private-flag enforcement (closes Track D follow-up)

- [ ] `Dht::announce(info_hash, private: bool)` rejects private torrents with a no-op. Regression test locked.
- [ ] Similarly for `get_peers` and the internal announce triggered by a torrent attach.

### G. Public API + engine integration

- [ ] `Dht::new(DhtConfig, Arc<UdpDemux>, cache_dir) -> Arc<Dht>`.
- [ ] `Dht::announce(info_hash) -> impl Future<Output = Result<Vec<SocketAddr>, DhtError>>`.
- [ ] `Engine::attach_dht(Arc<Dht>)` — torrent attach emits an async `get_peers` + `announce_peer`, discovered peers pumped into `Engine::add_peer`.

## Gate criteria

1. **All DISCIPLINES bars hold**: `cargo test`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo doc --no-deps -D warnings` clean. ADRs 0024–0026 land alongside the code.
2. **Three-engine DHT-only swarm (hard gate)**: `crates/magpie-bt-dht/tests/three_engine_swarm.rs` — three magpie engines on loopback, each with a DHT. A seeds, B + C bootstrap via A only (no trackers, no PEX). `announce` on B and C returns A's address; download completes with SHA-256 match. Silent-failure guard: the test asserts `Engine::add_peer` was invoked with exactly the DHT-learnt address (not a pre-configured peer).
3. **Cold-start bootstrap**: `tests/bootstrap_cold_start.rs` — an engine with an empty cache, backed by 4 mock DHT responders simulating the hardcoded bootstrap hosts, reaches the ≥32-good-nodes exit criterion within the 10-min stall window.
4. **Token round-trip**: unit test in `tokens.rs` confirms `make_token(ip, current)` validates; `make_token(ip, previous)` also validates; after two rotations the oldest token no longer validates; a token minted with a different IP is rejected.
5. **BEP 42 node ID**: unit test confirms generated ID passes BEP 42 validation for its derivation IP and fails for a different IP.
6. **Rate-limit correctness**: inbound flood at 200 QPS from a single source reaches 60-burst then drops remainder; global flood at 1000 QPS caps at 1500 burst then drops. `queries_dropped_rate_limited` counter increments on every drop.
7. **Private-flag regression**: unit test confirms `Dht::announce(hash, private=true)` returns Ok with an empty peer list and emits zero KRPC messages.
8. **ADRs landed**: 0024 (routing table), 0025 (bootstrap), 0026 (tokens + BEP 42 + rate limits).

## ADRs in this milestone

- [ADR-0024](../adr/0024-dht-routing-table.md) — routing table shape (split-on-demand binary tree, K=8, node-quality state machine)
- [ADR-0025](../adr/0025-dht-bootstrap.md) — bootstrap strategy (three-source contact list, zero-state flow, persistent cache)
- [ADR-0026](../adr/0026-dht-tokens-and-rate-limits.md) — tokens (stateless HMAC), BEP 42 node ID, three-tier rate limits

## Out of scope

- DHT-over-IPv6 (most deployed DHTs are v4-only; add when demand signals).
- Read-only mode (BEP 43). Low-priority add-on.
- DHT security extension BEP 42 *strict mode* on by default. Shipping disabled; consumers flip.
- BEP 44 mutable/immutable arbitrary-data storage. Out of scope; BT-swarm-discovery only.
- uTP and v2/BEP 52 remain M5.
